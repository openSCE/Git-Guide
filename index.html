  • filter_dramaStarting with Git
    • filter_dramaWhat is Git?
      Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

      Git is a version control system for tracking changes in computer files and coordinating work on those files among multiple people. It is primarily used for source code management in software development,but it can be used to keep track of changes in any set of files. As a distributed revision control system it is aimed at speed,data integrity,and support for distributed, non-linear workflows.
    • filter_dramaWhat is a repository?
      A repository is simply a place where all of your work along with the history of your work is stored.

      The purpose of Git is to manage a project, or a set of files, as they change over time. Git stores this information in a data structure called a repository. A git repository contains, among other things, the following: A set of commit objects. A set of references to commit objects, called heads.
    • filter_dramaWhat is fork and how to fork a repository?
      A fork is a copy of a repository. Forking a repository allows you to freely experiment with changes without affecting the original project. Most commonly, forks are used to either propose changes to someone else's project or to use someone else's project as a starting point for your own idea.

    • filter_dramaWhat is clone and how to clone a repository?
      The git clone command copies an existing Git repository. This is a “working copy”which has its own history, manages its own files, and is a completely isolated environment from the original repository.

      • filter_dramaCloning using Browser
        • On GitHub, navigate to the main page of the repository.

        • Under the repository name, click Clone or download



        • filter_dramaCloning usng Command Line
          • On GitHub, navigate to the main page of the repository.

          • Under the repository name, click Clone or download and copy the HTTPS link.

          • Or copy the link of that repository from the address bar.

          • In the Clone with HTTPs section, click to copy the clone URL for the repository.
          • Open Terminal.
          • Change the current working directory to the location where you want the cloned directory to be made.
          • Type git clone, and then paste the URL you copied.
          • For Example You copied: https://github.com/openSCE/Git-Guide then type the following command :
          • opensce@opensce-HP-Notebook:~$ git clone https://github.com/openSCE/Git-Guide
          • Press Enter. Your local clone will be created.
          • opensce@opensce-HP-Notebook:~$ git clone https://github.com/openSCE/Git-Guide
            Cloning into 'Git-Guide'...
            remote: Counting objects: 85, done.
            remote: Compressing objects: 100% (54/54), done.
            remote: Total 85 (delta 36), reused 78 (delta 29), pack-reused 0
            Unpacking objects: 100% (85/85), done.
            Checking connectivity... done.
            opensce@opensce-HP-Notebook:~$ |
          • A clone of that repository will be created at the present working directory.
    • filter_dramaAdding name to a remote repository?
      You can also give name to any remote repository so that you don't have to write link to the remote repository . You can simply use name instead of that link. For adding name to repository https://github.com/openSCE/Git-Guide write:
      opensce@opensce-HP-Notebook:~$ git remote add < name > < repository link >
      For Example :
      Name given is guide, you can give name of your choice if you want, the only thing to keep in mind is that the name should not be already used in the current project as alias to any remote link
      Repository link is https://github.com/openSCE/Git-Guide
      opensce@opensce-HP-Notebook:~$ git remote add guide https://github.com/openSCE/Git-Guide
      Now guide can be used instead of the link https://github.com/openSCE/Git-Guide.

      opensce@opensce-HP-Notebook:~$ git rebase -i 97c9d7d
